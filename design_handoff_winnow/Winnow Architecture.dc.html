How each operation works
Sync
Incremental deltas via users.history.list against a stored historyId. Headers first, bodies on open.
Send
messages.send for Gmail identities — honors verified send-as aliases & threading. SMTP for non-Gmail.
Search
Same q= syntax as Gmail web, so results match what users already know.
Push without a server
Gmail's instant push (Cloud Pub/Sub watch) needs a webhook — a server. Winnow polls history.list on foreground, a timer, and iOS background refresh instead. Slightly less instant, fully serverless.
Plan for: scope review
Restricted scopes (gmail.modify, https://mail.google.com/) require Google OAuth verification plus an annual CASA audit before public release — true for any Gmail client, either transport.
Tokens in the Keychain · requests from the device · no Winnow servers in the path — the privacy promise holds at the network layer, not just the UI.
